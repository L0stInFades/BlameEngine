§00

技术愿景：用真实代码塑造下一代模拟游戏

THE VISION · THE NEXT DIMENSION

传统的游戏引擎往往为了通用的商业目的妥协了极客特性。在需要高频代码运行、海量网络实体仿真和硬核 HackOps 玩法的场景下，我们必须提供一套全新设计的高性能运行时。

Blame Engine 提出了突破性的技术思路：将生产级别的编辑器环境直接集成至底层的 3D 渲染和 ECS 数据管线中。这意味着玩家的编写输入不用再经历漫长的解译，而是作为游戏世界的“第一等公民”立即执行。

  • 01

    极佳的终端渲染手感

    游戏视口即是真正的开发终端。通过手写的 msgpack-rpc 编解码器，将嵌入的 nvim 输入事件与状态画面直接映射为 GPU 渲染顶点，达成 0 延迟的极佳输入手感。

  • 02

    高可靠性安全沙箱

    沙箱与逻辑运行时的完全隔离。代码在底层可以安全、不受阻碍地异步执行，支持实时的语法报错诊断、LSP 关联提醒，以及复杂的多级逻辑回归。

  • 03

    毫秒级逻辑数据跃迁

    通过独立的多线程事件总线 (Event Bus)，代码的运行结果在几毫秒内即可转化为世界的状态修改，避免传统事件分发的死锁与堆栈延迟。

  • 04

    完美的模拟确定性

    引擎强制要求模拟更新（Simulation Tick）与帧渲染表现解耦。哪怕面临帧率大幅波动，模拟逻辑也能保持高度确定的时间步，支持逻辑的存档与完美回放。

创造超越想象的“真代码”交互体验。

我们不提供 SaaS 风味的模板化功能介绍。Unreal Engine 用无与伦比的写实画面赋能创作者，而 Blame Engine 则是用完全敞开的底层代码运行时赋能极客。以下是关于图形、流送、ECS 逻辑和架构的全部工程细节。

研发路线图与前沿挑战

TECHNICAL ROADMAP · COMPREHENSIVE DEVELOPMENT

要实现工业级的真代码安全执行游戏引擎，仍有几块重要的技术难关需要啃下。我们保持前沿研发过程的完全公开与透明。

  • engine/sandbox 沙箱隔离层 —— 当前策略直接由宿主 Python 进程或嵌入 Lua 环境执行，存在一定的开发机安全风险。后期计划将运行栈塞入隔离沙箱或轻量容器中。
  • World API 逻辑映射桥 —— 完善玩家程序读写大世界实体的统一句柄。目前是用“分值”作为间接路由映射，下个阶段将落地真正的双向内存映射桥。
  • Ops Workspace 控制面板 —— 支持复杂任务中的多策略多文件依赖协作，而不仅仅是单一文件的 Python 入侵钩子。
  • 终端视口完整字符渲染 —— 实现完整的浮动窗口（Floating Window）、外部 CMD 提示符、输入法（IME）的捕获与重定向，以及由 RHI 直接驱动的 C++ 画布缓冲刷新。
  • Windows 进程启动验证 —— POSIX 进程拉起在 macOS 平台上已经完备；Windows 的 CreateProcess 进程管道控制代码已落地，等待在真机环境中做持续测试。
  • 逻辑与大世界管线的交织 —— 确保流送线程的高吞吐读取与 ECS 并行任务的物理同步，不出现死锁或写读悬挂引用。
