# 0005. 渲染交给 UE5、物理用 Jolt;自研核心是"headless 权威世界 + 安全玩家代码运行时 + Game API"

- **Status**: Accepted
- **Date**: 2026-05-29
- **Supersedes/Adjusts**: 重定位 [ADR-0002](0002-archetype-ecs.md)(ECS = headless 世界核心);作废自研渲染路线(原 `02-rendering-roadmap.md` 已随渲染器删除归档至 [`../history/02-rendering-roadmap.md`](../history/02-rendering-roadmap.md))及 [`TECH_DEBT.md`](../TECH_DEBT.md) 中的自研渲染 P0。

## Context
全引擎交付就绪度审计的结论:自研渲染/内容工具/物理是**最大、最不差异化、最不可能做好**的工作(渲染器 ~28% 就绪、需专职渲染工程师、5+ 个月;内容工具数月;无物理)。而产品真正的护城河是**"真实代码黑客 + AI agent + 安全"在一个开放世界里**。从零自研开放世界引擎,等于重做 UE5/Unity 已解决的东西。

## Decision
1. **不自研渲染、不自研物理。**
   - **渲染 = Unreal Engine 5**,但**严格只当渲染/呈现客户端**:UE5 内**不写任何游戏逻辑、不持有权威游戏状态、不用 Blueprint/Gameplay Framework 当游戏逻辑**(Blueprint 至多用于美术纯表现/VFX/UI)。理由:若让 Blueprint 当游戏逻辑,会与"玩家写的真实代码"形成两个权威逻辑层,使玩家代码沦为二等公民,并摧毁安全/确定性/反作弊。
   - **物理 = Jolt**(权威、确定性、服务器侧)。
2. **自研、且公司拥有的核心 = headless 权威世界**:Archetype ECS(已有)+ Jolt + 世界状态 + 游戏规则 + 玩家代码运行时。**可在无渲染器下 headless 运行**——用于专用服务器、CI 测试、AI-agent 会话。
3. **玩家代码第一等公民、安全第一等公民**:玩家写的真实代码在 **WASM 能力沙箱**里运行(宿主只把 Game API 作为 import 暴露;燃料/内存限额;无逃逸),服务器权威。
4. **一套能力域化、可版本化、确定性的 Game API**,是**三个消费者的唯一契约**:玩家 WASM 代码、AI agent(读状态 + 限定写,帮玩家排任务/hack)、UE5 视图。
5. **sim↔UE5 边界 = headless 权威(方案 A)**:sim 是**可独立 headless 运行的库**;UE5 是很薄的 C++ 视图模块,只镜像被复制的状态(零游戏逻辑)。**首个垂直切片可把 sim 进程内嵌进 UE5 可执行体求速度,但边界按"网络/服务器权威"设计**,最终形态 = 专用权威服务器 + UE5 客户端。

## Consequences
- 正面:删掉最大/最不差异化的工作(渲染/内容/物理);把好钢用在护城河上;**渲染器可替换**(不锁死 UE5);headless 优先 → 核心可在无渲染器下开发/测试/被 AI-agent 使用;服务器权威 → 安全 + 反作弊 + 多人成为可能。
- 负面/新增工作(诚实):**sim↔UE5 状态复制层成为承重墙**;只用到约一半 UE5(渲染/动画/音频/内容/编辑器),另一半(玩法/世界/网络)自建;**UE5 许可成本**(Epic 5% 抽成 >100 万美元、随版本升级、非自有);需一条 review 红线"UE5 内无游戏逻辑";**确定性必须从一开始设计**。

## Alternatives considered
- 自研渲染器:否——数年、需专家、重做 UE5。
- UE5 当完整游戏框架 + Blueprint 写玩法:否——两个权威逻辑层,瓦解"玩家代码第一"与安全/确定性。
- 更轻的渲染前端(Godot,自有、无抽成):作为**后路**记录——若"把 UE5 掰成纯视图"的阻抗过大且画面无需顶级,可替换(本决策已把渲染器设计为可替换)。
- UE5 自带 Chaos 物理:否——选 Jolt 以获得独立于 UE5 的确定性服务器物理。
- 沙箱用 OS 级隔离(容器/microVM):备选——WASM 因"能力沙箱 + 多语言 + 偏确定性 + 轻量"更契合"玩家写真实代码 + 安全第一";重隔离需求出现时再叠加 microVM。
